# UmlTool

日本語 | [English](README.en.md)

Visio のように要素を自由配置・自由配線できる WYSIWYG の UML 作図ツール。
シーケンス図とアクティビティ図に対応。サーバーレスの Electron デスクトップアプリ。

- エンジン: [AntV X6](https://github.com/antvis/X6) v3 + Electron + TypeScript (electron-vite)
- テキスト（PlantUML サブセット）からの**初期生成は一方向**（生成後は GUI で自由に調整）
- プロジェクト保存は `.umlproj`（JSON エンベロープ + X6 モデル）
- 画像書き出し: PNG / JPEG / WebP

## 開発

```bash
npm install
npm run dev        # 開発起動
npm run typecheck  # 型チェック
npm test           # ユニットテスト (Vitest)
npm run build      # プロダクションビルド (out/)
npm start          # ビルド済みをプレビュー起動
```

### ヘッドレス自己診断

```powershell
$env:UMLTOOL_DIAG='1'; npx electron-vite preview
```

図の生成・3 形式書き出し・保存⇄読込ラウンドトリップ・編集挙動を検証し、
`diag-output.png`（アクティビティ図）/ `diag-output-seq.png`（シーケンス図）を出力します。

## 操作方法

| 操作 | 方法 |
|---|---|
| スクロール | マウスホイール（スクロールバーも可） |
| ズーム | Ctrl + ホイール（カーソル位置基準）、ツールバー −/100%/＋/全体 |
| パン | 空白部を左ドラッグ、または中ボタンドラッグ |
| 矩形選択 | Shift + 左ドラッグ |
| undo / redo | Ctrl+Z / Ctrl+Y（Ctrl+Shift+Z） |
| コピー / 切り取り / 貼り付け | Ctrl+C / Ctrl+X / Ctrl+V（貼り付けは少しずらして配置） |
| すべて選択 | Ctrl+A |
| 削除 | Delete / Backspace またはツールバー 🗑 |
| プロパティ編集 | 要素を選択 → 右パネルで名前・ラベル・種別を編集 |
| ラベル直接編集 | ノードやメッセージを**ダブルクリック**してその場で編集 |

ラベルはノードに合わせて自動で折り返され、ノード幅もラベルに合わせて自動調整されます（上限あり）。

### シーケンス図

- **メッセージの作成**: ライフラインの生存線（縦破線）からドラッグして相手のライフラインへ。
  またはツールバー「＋メッセージ」（2 要素を選択→選択順に接続。1 要素なら最寄りへ、未選択なら先頭 2 本）
- **メッセージの上下移動**: メッセージを選択し、中央のハンドルを上下ドラッグ（常に水平を維持）
- **繋ぎ替え**: 選択して端点ハンドルを別のライフラインへドラッグ
- **種別**: 同期（実線・塗り矢印）/ 非同期（実線・開矢印）/ 戻り（破線・開矢印）/ 自己（ループ）— 右パネルで切替
- **活性化バー（実行仕様）**: ツールバー「＋活性化バー」。ライフラインの中心線上に拘束され、上下移動・縦横リサイズ可

### アクティビティ図

- ツールバーの図種別セレクトで「アクティビティ図」に切替
- **フローの作成**: ノードにホバーすると出る接続ポート（上下左右の丸）からドラッグ。
  またはツールバー「＋フロー」（2 ノードを選択→選択順に接続。1 ノードなら最寄りへ）
- フローは直交ルーティング（manhattan）。選択して経由点を追加・調整可能
- ガード条件はフローを選択して右パネルで編集
- 新規ノードは**表示中の画面の中央**に追加される（連続追加時は少しずつずれる）

## テキスト生成（PlantUML サブセット）

### シーケンス図

```
participant ユーザー
actor 管理者
ユーザー -> サーバー : 同期メッセージ
ユーザー ->> サーバー : 非同期メッセージ
サーバー --> ユーザー : 戻り
サーバー -> サーバー : 自己メッセージ
```

### アクティビティ図

```
|受付|            ← スイムレーン（任意）
start
:注文を受け付ける;
if (在庫あり?) then (yes)
  :商品を引き当てる;
else (no)
  :取り寄せを手配する;
endif
fork
  :請求書を発行する;
fork again
  :商品を発送する;
end fork
stop
```

コメントは行頭 `'` `#` `//`。`@startuml` / `@enduml` / `title` は無視されます。

## ファイル形式 `.umlproj`

```json
{
  "format": "umltool-project",
  "version": 2,
  "diagramType": "sequence",
  "graph": { "cells": [] }
}
```

version 1（旧 maxGraph 実装の XML 形式）は読み込めません（明示エラー）。
