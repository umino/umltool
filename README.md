# UmlTool

日本語 | [English](README.en.md)

Visio のように要素を自由配置・自由配線できる WYSIWYG の UML 作図ツール。
シーケンス図とアクティビティ図に対応。サーバーレスの Electron デスクトップアプリ。

- エンジン: [AntV X6](https://github.com/antvis/X6) v3 + Electron + TypeScript (electron-vite)
- テキスト（PlantUML サブセット）からの**初期生成は一方向**（生成後は GUI で自由に調整）
- プロジェクト保存は `.umlproj`（JSON エンベロープ + X6 モデル）
- 画像書き出し: PNG / JPEG / WebP

## 開発

```bash
npm install
npm run dev        # 開発起動
npm run typecheck  # 型チェック
npm test           # ユニットテスト (Vitest)
npm run build      # プロダクションビルド (out/)
npm start          # ビルド済みをプレビュー起動
```

### ヘッドレス自己診断

```powershell
$env:UMLTOOL_DIAG='1'; npx electron-vite preview
```

図の生成・3 形式書き出し・保存⇄読込ラウンドトリップ・編集挙動を検証し、
`diag-output.png`（アクティビティ図）/ `diag-output-seq.png`（シーケンス図）を出力します。

## 操作方法

| 操作 | 方法 |
|---|---|
| スクロール | マウスホイール（スクロールバーも可） |
| ズーム | Ctrl + ホイール（カーソル位置基準）、ツールバー −/100%/＋/全体 |
| パン | 空白部を左ドラッグ、または中ボタンドラッグ |
| 矩形選択 | Shift + 左ドラッグ |
| 選択に追加 / 解除 | Ctrl + クリック |
| 複数まとめて移動 | 複数選択した状態で、いずれかをドラッグ |
| undo / redo | Ctrl+Z / Ctrl+Y（Ctrl+Shift+Z） |
| コピー / 切り取り / 貼り付け | Ctrl+C / Ctrl+X / Ctrl+V（貼り付けは少しずらして配置） |
| すべて選択 | Ctrl+A |
| 削除 | Delete / Backspace またはツールバー 🗑 |
| 部品の追加 | 左ペイン「部品」タブのアイコンをクリック |
| テキスト（付属） | 部品パレットの「テキスト」。選択中のライフラインに付属（破線で結び、移動に追従、シーケンス図）。内容はダブルクリック、フォントサイズ・太字・色は右パネル、ハンドルで横幅変更（自動折り返し） |
| ノート（付箋） | 部品パレットの「ノート」。左上折りの付箋で、どこにでも自由配置（両図種）。編集・スタイルはテキストと同様 |
| プロパティ編集 | 要素を選択 → 右パネルで名前・ラベル・種別を編集 |
| ラベル直接編集 | ノードやメッセージを**ダブルクリック**してその場で編集 |
| 外観の変更 | 要素を選択 → 右パネル「外観」で背景色・線の色・フォントサイズ・フォント・太字・文字色を変更 |

ラベルはノードに合わせて自動で折り返され、ノード幅もラベルに合わせて自動調整されます（上限あり）。フォントサイズやフォントを変えると、その実寸に合わせて自動調整もやり直されます。

### 外観（色・フォント）

- 要素を選択すると右パネルに「**外観**」欄が出ます。図形が持てる項目だけが表示されます（例: 開始・終了・フォークは背景色のみ、合流は背景色と線の色のみ）
- **カラーピッカーはプリセットと自由入力の併用**です。並んでいる色見本をクリックすれば即適用、左の色ボックスからは任意の色を選べます
- 対象: ライフライン / 活性化バー / フラグメント / アクション / 分岐 / 合流 / 開始 / 終了 / フォーク / ジョイン / レーン / フレーム / テキスト / ノート

### シーケンス図

- **メッセージの作成**: ライフラインの生存線（縦破線）からドラッグして相手のライフラインへ。
  または部品パレットの「メッセージ」（2 要素を選択→選択順に接続。1 要素なら最寄りへ、未選択なら先頭 2 本）
- **メッセージの上下移動**: メッセージを選択し、中央のハンドルを上下ドラッグ（常に水平を維持）
- **繋ぎ替え**: 選択して端点ハンドルを別のライフラインへドラッグ
- **種別**: 同期（実線・塗り矢印）/ 非同期（実線・開矢印）/ 戻り（破線・開矢印）/ 自己（ループ）— 右パネルで切替
- **活性化バー（実行仕様）**: 部品パレットの「活性化バー」。ライフラインの中心線上に拘束され、上下移動・縦横リサイズ可。バーが重なると、内側のものが astah* のように 1 段ずつ右へずれて積み重なります（ずれ幅はバー幅の半分）
- **複合フラグメント**: 部品パレットの「フラグメント」。種別（alt / opt / loop / break / par / seq / strict / critical）と条件は右パネル、条件はダブルクリックでも編集可。枠線ドラッグで移動、選択してリサイズ。alt / par は右パネル「＋区切り線を追加」で破線の区切りを追加（上下ドラッグで移動、Delete で削除）

### アクティビティ図

- ツールバーの図種別セレクトで「アクティビティ図」に切替
- **フローの作成**: ノードにホバーすると出る接続ポート（上下左右の丸）からドラッグ。
  または部品パレットの「フロー」（2 ノードを選択→選択順に接続。1 ノードなら最寄りへ）
- フローは直交ルーティング（manhattan）。選択して経由点を追加・調整可能
- ガード条件はフローを選択して右パネルで編集
- **分岐と合流は別ノード**: 分岐（デシジョン）は条件ラベル付きの菱形、合流（マージ）は小さな空の菱形（部品パレットの「合流」）
- **分岐の図形**: ツールバーの「分岐:」で **菱形** / **6 角形**（PlantUML 風の横長）を選べます。図全体の設定で、既存の分岐すべてと以降に追加する分岐に適用され、プロジェクトファイルに保存されます。合流は分岐と見分けが付くよう菱形のままです。切り替えるとレイアウトが崩れることがあります
- **枝は重ならない**: 分岐からは下・右・左へ、合流へは上・右・左から、枝が別々の辺に自動で振り分けられます（相手ノードの位置に応じて決まり、移動すると付け直されます）。枝が 4 本以上あって辺が足りないときだけ重なります
- **接続先は上辺中央**: 送信元が送信先より上にあるフローは、送信先の**上辺中央**に接続されます（上から下へ読める配線になります）。合流と分岐は上記の枝の振り分けが優先されます。**端点をドラッグで繋ぎ替えると、以降その端点は自動調整されません**（どのノードでも同じ）
- **スイムレーン**は選択するとハンドルが出て幅・高さを変更可能
- **ノードのリサイズ**: アクション / 分岐 / 合流 / 開始 / 終了 / フォーク / ジョインは選択するとハンドルが出て、右パネルの「幅」「高さ」でも変更可能（開始・終了は真円を保つよう縦横比固定）。アクションと分岐は通常ラベルに合わせて自動リサイズされるが、手動リサイズするとそのサイズが優先される。右パネルの「サイズを自動に戻す」で元の挙動に戻せる
- **フレーム（コンテナ）**: 部品パレットの「フレーム」。左上のヘッダタブ付きの透過枠で、中のノードはそのまま操作可能。枠線かヘッダをドラッグで移動、選択してリサイズ、ヘッダは右パネル/ダブルクリックで編集（タブ幅はテキストに追従）
- 新規ノードは**表示中の画面の中央**に追加される（連続追加時は少しずつずれる）

## テキスト生成（PlantUML サブセット）

### シーケンス図

```
participant ユーザー
participant "Web サーバー" as サーバー
actor 管理者
ユーザー -> サーバー : 同期メッセージ
ユーザー ->> サーバー : 非同期メッセージ
activate サーバー
サーバー --> ユーザー : 戻り
サーバー -> サーバー : 自己メッセージ
deactivate サーバー
alt 認証OK
  サーバー --> ユーザー : 成功
else 認証NG
  サーバー --> ユーザー : 失敗
end
note right of サーバー : ライフラインに付属するテキスト
note over ユーザー, サーバー : 二者にまたがる付箋
note left of ユーザー
複数行の
本文も書けます
end note
[-> サーバー : 図の外から来るメッセージ
サーバー ->] : 図の外へ出るメッセージ
autoactivate on
ユーザー -> サーバー : 呼び出しでバーが開く
サーバー --> ユーザー : 戻りでバーが閉じる
```

- **表示名とエイリアス**: `participant "表示名" as エイリアス`。以降はエイリアスで参照します。
- **活性化バー**: `activate 参加者` … `deactivate 参加者`。`deactivate` を省略すると末尾で自動的に閉じます。
- **バーの自動開閉**: `autoactivate on` 以降は、呼び出し（`->` / `->>`）が宛先のバーを開き、戻り（`-->`）が送信側のバーを閉じます。`autoactivate off` で以降のメッセージには適用されなくなります（開いたままのバーは末尾で閉じます）。明示の `activate` / `deactivate` と混在させても互いに影響しません。
- **複合フラグメント**: `alt` / `opt` / `loop` / `break` / `par` / `seq` / `strict` / `critical`（`end` で閉じる、ネスト可）。
  `else` による破線の区切りは `alt` / `par` 内で使えます（各オペランドの条件は生成後、右パネルでも個別に編集できます）。
- **注釈**: 2 種類の図形に振り分けられます。
  - `note left of 参加者` / `note right of 参加者` → **テキスト**（そのライフラインに付属し、破線で結ばれ、移動に追従）
  - `note over 参加者` / `note over 参加者, 参加者` → **ノート**（自由配置の付箋。複数指定するとその範囲を覆う幅になる）

  どちらも `:` 以降に本文を書くか、`:` を省いて次行から `end note` までを複数行の本文にできます。注釈はその直前のメッセージの下に置かれ、続くメッセージはその分だけ下にずれます。
- **図の外とのやりとり（ゲート）**: `[-> 参加者`（外から）と `参加者 ->]`（外へ）。矢印の種別（`->` / `->>` / `-->`）はそのまま使えます。ゲート側の端点は参加者を持たない自由な点になり、外から来るものは相手の左、外へ出るものは相手の右に伸びます。`[->]` のように両端を外にすることはできません。

### アクティビティ図

```
|受付|            ← スイムレーン（任意）
start
:注文を受け付ける;
if (在庫あり?) then (yes)
  :商品を引き当てる;
else (no)
  :取り寄せを手配する;
endif
fork
  :請求書を発行する;
fork again
  :商品を発送する;
end fork
stop
```

コメントは行頭 `'` `#` `//`。`@startuml` / `@enduml` / `title` は無視されます。

## ファイル形式 `.umlproj`

  "diagramType": "sequence",
  "graph": { "cells": [] }
